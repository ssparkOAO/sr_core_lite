SR Core Python Toolchain 建立紀錄

主題：TFLite 參數提取、golden 產生、Vivado hex 轉檔

回到：SR Core 最小功能設計紀錄

這份文件記錄目前 SR Core 專案使用的 Python 工具。目標是讓模型參數、golden tensor、 RTL testbench、Vivado BRAM 初始化檔之間有清楚分工。

1. 目前已建立的工具

工具 用途 輸出
extract_sr_tflite_params.py 從 TFLite INT8 model 抽出 weight、bias、scale、zero point、M0、M1。 generated/*.mem, generated/*.coe, sr_tflite_params.json
sr_core_golden.py 重建 runtime-verified quantized flow，產生逐層 golden tensor。 generated/golden/*.mem
tflite_flatbuffer.py 簡化 TFLite flatbuffer parser，供參數抽取工具使用。 共用 Python module
convert_generated_to_vivado_hex.py 把 signed decimal generated/*.mem 轉成 Vivado / BRAM 可用的 two's-complement hex。 generated_vivado_hex/*.memh, generated_vivado_hex/*.coe

2. 常用執行順序

一般重新產生資料時，建議順序如下：

python model_lite\sr_core\tools\extract_sr_tflite_params.py
python model_lite\sr_core\tools\sr_core_golden.py --height 8 --width 8 --seed 1
python model_lite\sr_core\tools\convert_generated_to_vivado_hex.py

前兩步建立 decimal parameter / golden；第三步把它們轉成 Vivado-friendly hex。

3. generated 與 generated_vivado_hex 的分工

資料夾 格式 用途
generated/ signed decimal text Python 檢查、Verilog testbench、人工閱讀與 debug。
generated_vivado_hex/ two's-complement hex .memh / .coe Vivado Block Memory Generator、BRAM init、$readmemh。

原則：generated/ 是 golden source；generated_vivado_hex/ 是由工具轉出來的硬體初始化格式，不手動修改。

6. 對 RTL / Vivado 的意義

RTL module verification 主要吃 generated/ 的 decimal golden。 Phase8.3 開始引入 Vivado BRAM IP 後，BRAM 初始化改吃 generated_vivado_hex/ 的 .coe。 

generated/*.mem
    ↓
convert_generated_to_vivado_hex.py
    ↓
generated_vivado_hex/*.coe
    ↓
Vivado Block Memory Generator

這樣可以避免把「人類看得懂的數值」和「硬體儲存的 bit pattern」混在同一個資料夾。
