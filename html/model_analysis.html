2. 模型架構

bilinearpp.py 定義的模型是非常小的單通道 x2 Super Resolution：

Input lr_input: [1, H, W, 1], float range 約 0..1
  ↓
Conv2D 3x3, Cin=1, Cout=8, padding=same, bias=True
  ↓
ReLU
  ↓
Conv2D 1x1, Cin=8, Cout=4, padding=same, bias=True
  ↓
PixelShuffle / DepthToSpace scale=2
  ↓
clipped ReLU, clamp 0..1
  ↓
Output: [1, 2H, 2W, 1]
對 FPGA 來說，這個模型很適合逐步驗證：只有一個 3x3 conv、一個 1x1 conv、一個 PixelShuffle、簡單 clamp，沒有 BatchNorm、Add residual、mask 或複雜 attention。

5. TFLite Ops 與 Activation

conversion log 顯示 TFLite ops：

CONV_2D, DEPTH_TO_SPACE, MINIMUM, QUANTIZE, RELU
Op意義硬體支援方式
QUANTIZEUINT8 input 轉成 internal INT8。q_s8 = q_u8 - 128，scale 依 TFLite 檔案。
CONV_2DConv1 3x3 與 Conv3 1x1。int8 × int8，int32 accumulate，int32 bias。
DEPTH_TO_SPACEPixelShuffle scale=2。使用既有 ref/pixel_sulffle address mapping。
MINIMUMclipped ReLU 的上限 clamp。限制最大值為 1.0 對應的 quantized value。
RELUclamp 下限 0。限制最小值為 zero point 對應的 0。

6. Quantized Convolution 硬體公式

TFLite int8 convolution 的基本概念如下。Activation zero point 可能是 -128，weight zero point 是 0。

acc_int32[co] =
  bias_int32[co]
  + Σ (input_int8 - input_zero_point) * weight_int8[co]

requantized =
  round(acc_int32 * input_scale * weight_scale[co] / output_scale)
  + output_zero_point

output_int8 =
  clamp(requantized, -128, 127)

Conv1 後需要 ReLU，因此 int8 output 下限要 clamp 到 output zero point：

if real_value < 0:
  output_int8 = output_zero_point

Final output 對外是 UINT8，因此最後一步需要從 internal INT8 轉回 UINT8：

uint8_output_real = (int8_internal - zp_internal) * scale_internal
uint8_q = round(uint8_output_real / output_scale) + output_zero_point

7. PixelShuffle 不可替代

模型最後使用的是 PixelShuffle / DepthToSpace。後續硬體必須保留這個排列方式，不應改成 Bilinear、Nearest Neighbor、Transposed Convolution 或 Deconvolution。

Conv3 output: [1,H,W,4]

每個位置有四個 channel:
  c0, c1, c2, c3

PixelShuffle scale=2:
  c0 → HR(2y,   2x)
  c1 → HR(2y,   2x+1)
  c2 → HR(2y+1, 2x)
  c3 → HR(2y+1, 2x+1)

8. FPGA 需要支援的基本運算

運算必要性第一版建議
UINT8 → INT8 轉換模型外部 input 是 uint8，內部 tensor 是 int8。先在 PL loader 或 PS 預處理做，需與 TFLite golden 一致。
3x3 window generationConv1 需要 same padding 的 3x3 window。參考 stream_window_3x3_cin8，先做小 patch。
int8 × int8 MACConv1/Conv3 核心運算。先支援小 Cout，int32 accumulate。
bias addTFLite Conv2D 有 int32 bias。每個 output channel 一個 bias。
per-channel requantizationweight 是 per-channel quantization。先用 PS 產生 multiplier/shift，PL 做 fixed-point multiply + shift。
ReLU / clampConv1 ReLU 與 final clipped ReLU。用比較器與 saturate。
PixelShuffle address mapping模型輸出放大 x2 的必要步驟。重用 ref/pixel_sulffle 設計。
Output pack/unpackPS/PL 透過 32-bit AXI stream 搬 byte。參考 connect.v。

9. 對逐步部署的意義

這個模型很小，因此適合用逐步驗證方式部署：

Stage 3A: 單層 Conv1 小 patch，先不做 PixelShuffle
Stage 3B: Conv1 + ReLU + requantization
Stage 3C: Conv3 1x1，產生 4-channel sub-pixel tensor
Stage 4A: Conv3 output 接 PixelShuffle，輸出 HR patch
Stage 4B: 加 final clamp 與 uint8 output
Stage 5: 完整 input image / patch，PS 與 TFLite golden 比對
