Phase8.3 Vivado BRAM GUI 操作紀錄

文件位置：model_lite/sr_core/html/phase8_3_vivado_bram_gui.html

這份文件專門記錄 Vivado GUI 如何重建 Phase8.3 的 Block Memory Generator ROM IP。 目標是未來只看這份文件，就能用 GUI 做出和 Tcl 相同的 IP 設定。

步驟1：IP Catalog

Vivado
    ↓
IP Catalog
    ↓
Block Memory Generator
GUI 選項 實際選擇值 為什麼選這個值 對應 Tcl
IP Block Memory Generator 用 Xilinx 官方 IP 產生 BRAM/ROM，之後比較容易接近 Vivado deployment flow。 create_ip -name blk_mem_gen -vendor xilinx.com -library ip
Project sr_core_bram_pynqz2 把 BRAM prototype 和原本 verified RTL 分開，避免污染 RTL/ golden version。 create_project sr_core_bram_pynqz2 ... -part xc7z020clg400-1

步驟2：Customize IP 共通設定

六個參數 IP 都使用同一類型的 ROM 設定，只是 width/depth/COE 不同。

GUI 選項 實際選擇值 為什麼選這個值 對應 Tcl
Memory Type Single Port ROM 目前 weight/M0/M1 是固定參數，只需要讀，不需要 runtime 寫入。 set_property CONFIG.Memory_Type {Single_Port_ROM}
Port Type Native / Port A 目前只需要簡單 address/read data 介面，不引入 AXI 或 controller。 BMG native port default
Enable Port Type Use ENA Pin 保留 read enable，未來 controller 可以明確控制何時讀參數；若要 always read，可直接把 ena 接 1'b1。 set_property CONFIG.Enable_A {Use_ENA_Pin}
Load Init File true 讓 BRAM ROM 直接由 COE 初始化，符合目前固定參數 prototype。 set_property CONFIG.Load_Init_File {true}
COE File generated_vivado_hex/*.coe 用 fixed-width two's-complement hex 保存 signed parameter bit pattern，並集中管理 Vivado 用初始化檔。 set_property CONFIG.Coe_File {...}
Output Register disabled 先減少額外 latency，讓 prototype timing 較容易對照。未來若要提高 Fmax 可再打開。 CONFIG.Register_PortA_Output_of_Memory_Primitives false
Reset Port disabled ROM read data 不需要另外 reset，避免多餘控制腳位。 set_property CONFIG.Use_RSTA_Pin {false}

步驟3：各 IP 的 Width / Depth

IP 名稱 Read Width Read Depth COE 為什麼這樣配置
conv1_weight_rom 8 72 generated_vivado_hex/conv1_weight.coe Conv1 是 8 個 output channel，每個 channel 有 3x3 共 9 個 weight，所以 8*9=72。
conv3_weight_rom 8 32 generated_vivado_hex/conv3_weight.coe Conv3 是 4 個 output channel，每個 output channel 讀 8 個 input channel weight，所以 4*8=32。
conv1_m0_rom 32 8 generated_vivado_hex/conv1_m0.coe Conv1 有 8 個 output channel，每個 channel 一個 M0。
conv1_m1_rom 64 8 generated_vivado_hex/conv1_m1.coe Conv1 每個 output channel 一個 64-bit M1。
conv3_m0_rom 32 4 generated_vivado_hex/conv3_m0.coe Conv3 有 4 個 output channel，每個 channel 一個 M0。
conv3_m1_rom 64 4 generated_vivado_hex/conv3_m1.coe Conv3 每個 output channel 一個 64-bit M1。

步驟4：對應 Tcl 範例

create_ip -name blk_mem_gen -vendor xilinx.com -library ip -module_name conv1_weight_rom

set_property -dict [list \
  CONFIG.Memory_Type {Single_Port_ROM} \
  CONFIG.Write_Width_A {8} \
  CONFIG.Read_Width_A {8} \
  CONFIG.Write_Depth_A {72} \
  CONFIG.Load_Init_File {true} \
  CONFIG.Coe_File {generated_vivado_hex/conv1_weight.coe} \
  CONFIG.Enable_A {Use_ENA_Pin} \
  CONFIG.Use_RSTA_Pin {false} \
] [get_ips conv1_weight_rom]

實際 Tcl 寫在 RTL_sys/phase8_3_vivado_bram/tcl/create_sr_core_bram_pynqz2.tcl。 其中 Load_Init_File 和 Coe_File 要一起設定，避免 Vivado 在還沒有 COE 時先驗證失敗。

目前學到的重點

GUI 上看似只是填 width/depth/COE，但真正關鍵是：signed parameter 必須先轉成正確 bit pattern， 而且 BRAM read 是 synchronous timing。未來把它接回 CNN datapath 時，不能再用 reg array 的 combinational 直覺來看。
