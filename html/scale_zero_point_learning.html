Scale 與 Zero Point 學習紀錄

主題：為什麼量化是 q = round(r / S) + Z，而不是 q = round((r + Z) / S)

回到：SR Core 最小功能設計紀錄

1. 這次學到的核心觀念

量化公式中，S 和 Z 負責的是兩件不同的事情。

Scale S 決定每一格代表多少真實數值，也就是比例。 Zero point Z 決定真實世界的 0 要放在整數座標的哪裡，也就是座標偏移。 

正確公式:

q = round(r / S) + Z

反量化:

r = S * (q - Z)

直覺上，要先用 S 決定「這個 real value 是第幾格」， 再用 Z 決定「整數座標的原點在哪裡」。

2. S 和 Z 的角色分工

符號 角色 直覺 單位
S scale 每一格有多寬 真實值 / 整數格
Z zero point 真實世界的 0 對應到哪一個整數座標 整數座標
r real value 真實世界的數值 真實值
q quantized value 硬體裡存的整數 整數座標

因為 Z 是整數座標偏移，不是真實世界的數值， 所以不能拿 r + Z 這種方式先加在一起。

3. 用 -1 到 1 的範圍理解

先看真實世界的數值範圍： 

real r:

-1              0              1
 |--------------|--------------|

假設要對應到 int8 的 256 個格子： 

int8 q:

-128                         127
  |---------------------------|

若 real range 是 [-1, 1]，那 scale 可以理解成： 

S = (beta - alpha) / 255
  = (1 - (-1)) / 255
  = 2 / 255
  = 0.00784

也就是每一個整數格代表約 0.00784 的真實數值。 這一步只是在切尺，還沒有決定真實世界的 0 放在哪個 int8 位置。

4. 正確順序：先縮放，再平移

假設： 

r = 0.2
S = 2 / 255
Z = -128

先除以 scale： 

r / S = 0.2 / (2 / 255)
      = 25.5

意思是：0.2 這個真實值大約落在第 25.5 格。 

real:

0.2
 ↓

先用 S 決定第幾格:

25.5 → round → 25

接著再加 zero point： 

q = 25 + (-128)
  = -103

所以： 

真實值 0.2
  ↓ 除以 S，決定第幾格
25
  ↓ 加 Z，決定整數座標原點
-103

5. 錯誤順序：先加 Z 再除 S

如果錯誤地寫成： 

q = round((r + Z) / S)

代入同樣例子： 

r = 0.2
Z = -128
S = 2 / 255

(r + Z) = 0.2 - 128
        = -127.8

(r + Z) / S = -127.8 / (2 / 255)
            = -16294.5

這個結果完全沒有物理意義，因為 Z=-128 不是「真實世界的 -128」， 它只是「整數座標的 -128」。

錯誤的根本原因：把不同單位的東西加在一起。 r 是真實值，Z 是整數座標，它們不能直接相加。

6. 用 FPGA 影像輸入理解

假設影像 pixel 是 uint8，範圍 0..255。 若模型 input 是 int8，常見設定接近： 

input scale = 1 / 255 = 0.00392
input zero point = -128

對 pixel 100 來說，如果把它視為真實值 100/255， 量化到 int8 input 等價於： 

q = round((100 / 255) / (1 / 255)) + (-128)
  = 100 - 128
  = -28

所以實作上常可以直接寫： 

input_int8 = pixel_uint8 - 128

但這成立的前提是：uint8 pixel 的 real scale 和模型 input scale 對得上。 若 scale 不同，就不能只做簡單減 128，必須依公式轉換。

7. 圖像化流程

正確流程：

真實值 r
  ↓
[除以 S]
  ↓
決定第幾格
  ↓
[加 Z]
  ↓
整數座標 q

錯誤流程：

真實值 r
  ↓
[先加 Z]
  ↓
把真實值和整數座標混在一起
  ↓
[再除以 S]
  ↓
數值爆掉，沒有物理意義

8. 對 SR Core 的意義

對目前 SR FPGA project 來說，這個觀念會影響三個地方：

位置 要做的事 原因
PS input preprocessing 把 uint8 image 轉成模型 input tensor 需要的 quantized value。 input scale / zero point 決定第一層 Conv 吃到什麼 int8。
PL Conv requantization 用 M0/M1 把 int32 accumulation 轉成下一層 int8。 每一層 output scale / zero point 可能不同。
final output 把最後 int8 / uint8 output 對回影像數值。 輸出給 PS 比對 TFLite golden 或顯示影像。

簡單記法：S 是尺的刻度，Z 是尺的零點位置。 先看數值落在第幾格，再把座標原點移到對的位置。
