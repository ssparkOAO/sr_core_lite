文件位置：model_lite/sr_core/html/RTL_sys_index.html

工作區：model_lite/sr_core/RTL_sys/

Accelerator System Architecture Workspace。這裡放的是 Phase7 end-to-end PASS 之後的 system architecture refactor，不直接修改 RTL/ verified module library。

1. Workspace 定位

區域 定位 狀態
Module RTL 已驗證通過的 module library，保留在 model_lite/sr_core/RTL/。 Verified
System RTL 新的 architecture refactor 工作區，包含 weight memory、streaming、BRAM、AXI、DMA。 Developing
Deployment 未來的 Vivado block design、PS-PL integration、完整 accelerator deployment。 Future

規則：RTL/ 是 Golden RTL Version；RTL_sys/ 是新的 system architecture workspace。 新的架構重構不要直接改 verified RTL。

4. Phase8.3 目前重點

Phase8.3 把 Phase8 的 memory-style parameter organization 推進到 Vivado Block Memory Generator IP。 目前已建立六個 ROM IP prototype： 

conv1_weight_rom
conv3_weight_rom
conv1_m0_rom
conv1_m1_rom
conv3_m0_rom
conv3_m1_rom

目前仍不做 AXI、DMA、PS-PL。下一步才會研究如何把 parameter ROM bank 接回 top wrapper， 並處理 synchronous read latency。

5. 開發規則

  • 不要修改 model_lite/sr_core/RTL/ 內的 verified modules。
  • 新的 architecture refactor 放在 model_lite/sr_core/RTL_sys/。
  • 每個 Phase 都要同步建立 HTML 文件，記錄設計理由、驗證方式、遇到的問題、下一步。
  • 中文 HTML 一律保留 <meta charset="utf-8">，避免整檔 PowerShell rewrite。
