Phase8.4 ROM-backed Wrapper 分析

文件位置：model_lite/sr_core/html/phase8_4_rom_backed_wrapper.html

Vivado project：model_lite/sr_core/vivado/sr_core_bram_pynqz2

分析報告：model_lite/sr_core/RTL_sys/phase8_3_vivado_bram/sim/param_rom_ip_config_report.txt

這份文件是 Phase8.4 的第一步：先分析目前 Vivado project 中 6 個 Block Memory Generator ROM IP 的配置。 本階段尚未開始 wrapper integration。

1. 本階段目標

Phase8.3 已驗證 Vivado ROM IP 可以正確 readback 參數。Phase8.4 的下一步會把 ROM IP 接回 memory-oriented top wrapper，確認整個 SR pipeline 最終 output_uint8 仍然 PASS。

但在接 wrapper 前，必須先確認 ROM IP 的實際配置，尤其是 read latency、enable、output register。 這些會直接影響後續 address scheduling 與 valid timing。

3. 各欄位說明

3.1 ROM Type

六個 IP 都是 Single_Port_ROM。這符合目前用途：weight / M0 / M1 是固定參數， inference 過程只需要讀，不需要寫。

3.2 Data Width

資料 Width 原因
Conv1 / Conv3 weight 8 INT8 signed weight。
M0 32 requant multiplier 使用 signed 32-bit。
M1 64 requant offset 使用 signed 64-bit。

3.3 Address Width

Address width 由 depth 推導：ceil(log2(depth))。 例如 Conv1 weight depth 是 72，需要 7-bit address，因為 6-bit 只能表示 64 筆。 

conv1_weight_rom: depth 72 -> addr width 7
conv3_weight_rom: depth 32 -> addr width 5
conv1_m0/m1_rom:  depth 8  -> addr width 3
conv3_m0/m1_rom:  depth 4  -> addr width 2

3.4 Read Latency

Output register 雖然關閉，但 BMG ROM 仍是 synchronous read。也就是 address / enable 在某個 clock edge 被取樣後，資料在下一個 clock 週期可用。後續 wrapper 不能把 ROM 當成 combinational lookup table。

後續 Phase8.4 wrapper integration 的核心風險不是資料內容，而是 read latency。 Controller 必須多留一拍，或在啟動前先 preload 參數。

3.5 Enable Signal

六個 IP 都保留 ena，Vivado 設定為 Use_ENA_Pin。 如果設計每一拍都要讀，可以把 ena tie high；如果後續有 controller，則可以用 ena 標示有效讀取週期。

3.6 Output Register

目前 Register_PortA_Output_of_Memory_Primitives 與 Register_PortA_Output_of_Memory_Core 都是 false。 這代表沒有額外 output pipeline register。若未來打開 output register，read latency 會再增加， 但 timing 可能較容易收斂。

5. 對 Wrapper Integration 的影響

Phase8.2 的 memory-oriented wrapper 目前使用 internal reg memory，邏輯上比較像可以直接取值。 Phase8.4 若改成 ROM-backed wrapper，需要處理以下差異：

  • ROM read data 不是 combinational，至少要等 1 clock。
  • 原本一次展開所有 weight port 的方式，未來可能要改成 preload register bank。
  • 若仍要餵給既有 verified module 的大量 port，最簡單方式是 reset/start 前先從 ROM 讀出所有參數並暫存在 wrapper registers。
  • 若要真正 streaming / on-demand read，則要重新設計 MAC datapath 的 weight access timing，這會比較像 Phase9 之後的工作。

Phase8.4 建議先走保守路線：ROM IP -> preload registers -> existing verified top ports。 這樣可以先驗證 top 換成 ROM-backed parameter source 後，end-to-end output 仍然 PASS。
