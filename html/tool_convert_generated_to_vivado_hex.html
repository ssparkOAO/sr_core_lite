工具位置：model_lite/sr_core/tools/convert_generated_to_vivado_hex.py

輸出位置：model_lite/sr_core/generated_vivado_hex/

回到：SR Core 最小功能設計紀錄

這個工具負責把 generated/ 內的 signed decimal .mem 轉成 Vivado / BRAM 友善的 two's-complement hex 檔案。

1. 為什麼需要這個工具

generated/ 內的檔案主要是給 Python golden flow 與 Verilog testbench 使用， 格式是 signed decimal text，例如： 

63
-32
31
-26

但是 Vivado Block Memory Generator 比較適合吃明確的 bit pattern。 所以負數不能只看成「-32」這個文字，而要轉成固定 bit width 的 two's-complement hex。 

-32 as signed int8  -> E0
-1  as signed int32 -> FFFFFFFF
-1  as signed int64 -> FFFFFFFFFFFFFFFF

2. 什麼時候要執行

只要重新產生或修改以下任一資料，就應該重新執行一次：

  • generated/conv1_weight.mem
  • generated/conv3_weight.mem
  • generated/conv1_m0.mem / generated/conv1_m1.mem
  • generated/conv3_m0.mem / generated/conv3_m1.mem
  • generated/golden/*.mem

簡單規則：只要 generated/ 有更新，就重跑這個工具，讓 generated_vivado_hex/ 跟著同步。

3. 執行方式

請在專案根目錄執行：

cd C:\M11413047\codex\final_proj
python model_lite\sr_core\tools\convert_generated_to_vivado_hex.py

執行完成後會產生或更新：

model_lite/sr_core/generated_vivado_hex/

4. 輸出內容

每個來源 .mem 會產生兩種檔案：

副檔名 用途 範例
.memh 給 Verilog $readmemh 或 behavioral BRAM model 使用。 conv1_weight.memh
.coe 給 Vivado Block Memory Generator 的 Load Init File 使用。 conv1_weight.coe

輸出會保留原本子資料夾結構：

generated/golden/output_uint8.mem
    ↓
generated_vivado_hex/golden/output_uint8.memh
generated_vivado_hex/golden/output_uint8.coe

5. 位寬與 signed 規則

檔案類型 Width Signed 原因
*weight.mem 8 Yes INT8 weight。
*int8*.mem 8 Yes 中間 tensor 是 signed int8。
*uint8*.mem 8 No input / final output image domain 是 uint8。
*acc_int32.mem, *bias.mem, *m0.mem 32 Yes accumulator、bias、M0 都用 32-bit signed 解讀。
*m1.mem 64 Yes M1 是 requant offset，使用 signed 64-bit。

6. 驗證方式

工具不是只負責轉檔，也會自己做 round-trip verification： 

decimal value
    ↓
two's-complement hex
    ↓
依照 width / signed 解回 decimal
    ↓
compare original decimal

驗證結果會寫到：

model_lite/sr_core/generated_vivado_hex/conversion_verify_result.txt

目前結果：

total files=16
total values=2316
total mismatch=0
PASS

7. 和 Phase8.3 的關係

Phase8.3 的 Vivado BRAM prototype 會使用 generated_vivado_hex/*.coe 作為 Block Memory Generator 的初始化檔。例如： 

conv1_weight_bram -> generated_vivado_hex/conv1_weight.coe
conv3_weight_bram -> generated_vivado_hex/conv3_weight.coe
conv1_m0_bram     -> generated_vivado_hex/conv1_m0.coe
conv1_m1_bram     -> generated_vivado_hex/conv1_m1.coe
conv3_m0_bram     -> generated_vivado_hex/conv3_m0.coe
conv3_m1_bram     -> generated_vivado_hex/conv3_m1.coe

這樣 generated/ 保留人類容易檢查的 decimal golden， generated_vivado_hex/ 則專門服務 Vivado / BRAM / $readmemh。

8. 注意事項

  • 不要手動改 generated_vivado_hex/，需要更新時重跑 Python 工具。
  • 如果模型、QAT 參數、golden tensor 有重新產生，也要重跑此工具。
  • Vivado BRAM 看到的是 bit pattern，RTL 端要用正確 signed / unsigned 宣告解讀。
  • conversion_verify_result.txt 必須是 PASS 才能拿去餵 BRAM IP。
