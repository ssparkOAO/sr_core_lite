Phase9 Project Split Guide

文件位置：model_lite/sr_core/html/phase9_project_split_guide.html

這份文件用來區分：哪些檔案只是 behavioral RAM/ROM model 驗證， 哪些檔案才是真正透過 Vivado BMG IP wrapper 驗證。

1. 一句話分辨

工作區 代表意義 是否真的測 Vivado IP
RTL_sys/phase9_3_ref_hier/ Reference-like hierarchy + behavioral RAM/ROM model 否
RTL_sys/phase9_4_ip_verify/ + vivado/sr_core_streaming_pynqz2/ 同一個 sr_nn_top 接 Vivado BMG ROM/RAM IP wrapper 是

2. Phase9.3 資料夾怎麼看

RTL_sys/phase9_3_ref_hier/
├─ rtl/       真正的 Phase9.3 hierarchy RTL
├─ sim/       behavioral RAM/ROM model 與 TB
├─ results/   behavioral PASS 結果
└─ work/      xsim log、snapshot、.wdb 波形資料

work/ 裡看到的 .wdb 是 Vivado/xsim 波形資料庫。 它不是 Verilog source，也不是你要修改的地方。若要看波形，才用 Vivado Simulator 開啟。

3. Phase9.4 資料夾怎麼看

RTL_sys/phase9_4_ip_verify/
├─ sim/       IP-backed testbench
└─ results/   IP-backed PASS 結果與 dump

Phase9.4 的 RTL 仍然沿用 phase9_3_ref_hier/rtl/ 的 sr_nn_top。差異在於 simulation memory 不再使用 ram_ip_bhv.v，而是使用 Vivado project 裡的 BMG IP wrapper。 目前 project 設定為 target_language = Verilog、 simulator_language = Verilog，IP simulation wrapper 使用 *.v。

4. Vivado Project 現在保留什麼

vivado/sr_core_streaming_pynqz2 是 Phase9.4 的主要 Vivado project。 Project source 應保留：

  • RTL_sys/phase9_3_ref_hier/rtl/*.v
  • RTL/ 內已 verified modules
  • 8 個 Vivado BMG IP：6 個 parameter ROM、2 個 feature/output RAM
  • RTL_sys/phase9_4_ip_verify/sim/tb_sr_nn_top_ip.v

Project 內不應再保留舊的 Phase9.2 TB、Phase9 behavioral RAM file、 或 Phase8 parameter ROM bank 草稿。

5. 如何重新跑 IP-backed 驗證

C:\Xilinx\Vivado\2023.1\bin\vivado.bat -mode batch -source model_lite/sr_core/vivado/sr_core_streaming_pynqz2/tcl/clean_phase9_4_project.tcl
C:\Xilinx\Vivado\2023.1\bin\vivado.bat -mode batch -source model_lite/sr_core/vivado/sr_core_streaming_pynqz2/tcl/run_phase9_4_ip_sim.tcl

第一行整理 Vivado project source membership；第二行執行真正 BMG IP-backed simulation。

若要確認 project/IP wrapper language： 

C:\Xilinx\Vivado\2023.1\bin\vivado.bat -mode batch -source model_lite/sr_core/vivado/sr_core_streaming_pynqz2/tcl/report_ip_language.tcl

6. 常見 Vivado Warning

訊息 意義 目前是否影響 PASS
No write access right to the local Tcl store Vivado 沒辦法寫入本機 Tcl store，所以退回安裝目錄。 不影響本專案模擬。
Unable to auto find GCC executables XSim 找不到 GCC path。此設計目前沒有用 DPI/C 混合模擬。 不影響本專案模擬。
BMG is using a behavioral model for simulation 這是 Vivado BMG IP 自己提供的 simulation model，不是我們的 ram_ip_bhv.v。 正常；代表正在用 IP wrapper 做模擬。

7. What Did We Learn?

Phase9.3 是「設計形狀」驗證，Phase9.4 是「Vivado IP 記憶體」驗證。 從現在開始，只要你想確認是否真的用 IP，請看 Phase9.4 與 sr_core_streaming_pynqz2.xpr，不要只看 Phase9.3 的 behavioral PASS。
