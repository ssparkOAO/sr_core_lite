Phase2.5 Conv3 1x1 Block

本頁記錄 Conv3 layer 的 small pipeline： sr_conv1x1_cin8_cout4_mac 接上 4 個 sr_requantize，形成 sr_conv1x1_cin8_cout4_block。

1. Architecture

命名注意：Conv3 是 layer 名稱，kernel 是 1x1。 這一頁做的是 Conv3 的 1x1 MAC + requant，不是 Conv1 的 3x3 kernel。 

conv1_out_int8
  -> sr_conv1x1_cin8_cout4_mac
  -> acc0, acc1, acc2, acc3
  -> 4x sr_requantize
  -> conv3_out_int8

這個 block 保持 module 分離：MAC 還是 MAC，requant 還是共用的 sr_requantize。後面 Conv1 / Conv3 都可以重複使用同一個 requant block。

2. Tensor Mapping

Tensor Shape Flatten mapping
conv1_out_int8.mem (H,W,8) NHWC。每個 pixel 連續 8 個 channel：c0~c7。
conv3_weight.mem [4,1,1,8] cout-major。w00~w07 屬於 output channel 0。
conv3_out_int8.mem (H,W,4) NHWC。每個 pixel 連續 4 個 output channel：q0~q3。

3. Requant Flow

每個 output channel 使用各自的 M0 與 M1：

q0 = sr_requantize(acc0, conv3_m0[0], conv3_m1[0])
q1 = sr_requantize(acc1, conv3_m0[1], conv3_m1[1])
q2 = sr_requantize(acc2, conv3_m0[2], conv3_m1[2])
q3 = sr_requantize(acc3, conv3_m0[3], conv3_m1[3])

目前 requant formula：

tmp = acc * M0 + M1
rounded = tmp >>> 32
if tmp[31] == 1:
    rounded = rounded + 1
q_out = clamp(rounded, -128, 127)

4. Verification Flow

Testbench 每次取一個 pixel 的 8-channel input，送進 block，再比對 4-channel final int8 output。

for each pixel:
    input  = conv1_out_int8[pixel][0..7]
    weight = conv3_weight[0..3][0..7]
    M0/M1  = conv3_m0 / conv3_m1
    rtl    = q0, q1, q2, q3
    golden = conv3_out_int8[pixel][0..3]

若 mismatch，testbench 會印出：

pixel index
output channel
RTL value
golden value

5. Memory Mapping

使用檔案：

generated/golden/conv1_out_int8.mem
generated/conv3_weight.mem
generated/conv3_m0.mem
generated/conv3_m1.mem
generated/golden/conv3_out_int8.mem

目前這些 .mem 實際內容是 signed decimal text，不是 hex。 所以 testbench 使用 $fscanf(..., "%d") 讀取，避免負數讀錯。
