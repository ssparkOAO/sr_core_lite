1. 目前模型狀態

目前模型：

Bilinear_PP_v5_test_qat.tflite

模型資料流：

input_uint8
  -> QUANTIZE
  -> input_int8_for_core
  -> Conv1 3x3 (Cin=1, Cout=8)
  -> Requant + ReLU
  -> Conv3 1x1 (Cin=8, Cout=4)
  -> Requant
  -> PixelShuffle x2
  -> output_uint8

命名注意：Conv1 是模型中的第一個 convolution layer， 它的 kernel 是 3x3。Conv3 是模型後面的 convolution layer， 它的 kernel 是 1x1。所以 conv3 這個名字不是代表 3x3 kernel。

3. 目前專案策略

目前不做：

AXI
DMA
PS-PL
full SoC integration
runtime camera input

目前專案階段：

純 PL / RTL quantized compute core verification

現在的重點是逐層 module verification，不是一次完成 whole accelerator。 這樣可以把 arithmetic、dataflow、buffer mapping 分開確認， 每一關都用 runtime-verified golden 做 bit-exact compare。

4. 最終 RTL 規劃

最終 RTL SR Core 的資料流：

input_int8_for_core
  -> Conv1 3x3 MAC
  -> Requant + ReLU
  -> Conv3 1x1 MAC
  -> Requant
  -> PixelShuffle
  -> output_uint8

規劃 RTL modules：

Module 功能 目前狀態
sr_window_3x3 產生 Conv1 需要的 3x3 window。 Phase4 已完成 sr_window_3x3_cin1
sr_conv3x3_cin1_cout8_mac Conv1 3x3 MAC，Cin=1，Cout=8。 Phase4 已完成，conv1_acc_int32 PASS
sr_requantize fixed-point requantization arithmetic。 已完成第一版，Conv3 layer 1x1 output requant PASS
sr_activation_relu_int8 Conv1 requant 後的 int8 ReLU。 待做
sr_conv1x1_cin8_cout4_mac Conv3 layer 的 1x1 MAC，Cin=8，Cout=4。 已完成第一版，Conv3 layer 1x1 MAC PASS
pixel_shuffle_core 產生 PixelShuffle x2 write transaction。 已完成 transaction core，pixel_shuffle_out_int8 PASS
sr_core_top 整合各 module，形成完整 SR compute core。 待做
sr_output_stage Final output formatting: signed int8 clip + uint8 conversion。 Phase6 已完成，output_uint8 PASS
sr_output_clip PixelShuffle 後的 signed int8 output range constraint。 Phase5 已完成，clamp cases PASS
sr_conv1_3x3_cin1_cout8_block Conv1 3x3 window + MAC + 8x sr_requantize。 Phase4.5 已完成，conv1_out_int8 PASS
sr_conv1x1_cin8_cout4_block Conv3 layer 1x1 MAC + 4 個 sr_requantize。 已完成第一版，conv3_out_int8 PASS

5. 目前開發策略

由小到大
先 arithmetic，再 dataflow
先 module，再 top

原因是目前 golden 已完整，所以最適合逐 module 與 golden compare。 先確認每個小 module 都 bit-exact，再逐步往 dataflow 和 top-level 整合。
