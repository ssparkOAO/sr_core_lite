4. Frozen Clean-Stream Snapshot Verilog

Phase11.1 不直接改 frozen source，而是引用 PS_PL clean-stream Vivado snapshot： model_lite/sr_core/PS_PL/vivado/sr_core_clean_stream_pynqz2/src/rtl/。

Module / File 用途
sr_top_clean_stream_img.v Clean-stream SR core top。整合 parameter preload、Conv1、Conv3、PixelShuffle、OutputStage、output RAM。
sr_ctrl_clean_stream_img.v Clean-stream controller。控制 input stream、core run、PixelShuffle / output RAM 流程。
sr_pctrl.v Parameter preload controller。從 ROM 讀取 weight / M0 / M1，整理成 flat parameter bank。
sr_conv1_3x3_cin1_cout8_flat.v Conv1 full block。3x3 window + Cin=1/Cout=8 MAC + requant，使用 flat parameter bus。
sr_window_3x3_cin1.v Conv1 3x3 sliding window，padding 使用 input zero point。
sr_conv3x3_cin1_cout8_mac.v Conv1 3x3 signed int8 MAC，輸出 int32 accumulator。
sr_conv1_to_conv3_stream_slice.v Conv1 -> Conv3 streaming register slice。移除 feature_mem0 後負責 valid/data 對齊。
sr_conv1x1_cin8_cout4_flat.v Conv3 full block。1x1 Cin=8/Cout=4 MAC + requant，使用 flat parameter bus。
sr_conv1x1_cin8_cout4_mac.v Conv3 1x1 signed int8 MAC，輸出 4 個 int32 accumulator。
sr_requantize.v TFLite-style fixed-point requantization，共用於 Conv1 與 Conv3。
pixel_shuffle_core.v PixelShuffle x2 write transaction generator。產生 2x2 HR block 的 packed write。
sr_output_pack2x2_uint8.v 把 PixelShuffle 產生的 2x2 signed int8 block 轉成 uint8 並打包給 output RAM。
sr_output_stage.v Final output stage。signed int8 clip 後做 +128 轉 uint8。

5. Vivado Block Design IP

IP Vivado cell / module 用途
ZYNQ7 Processing System zynq_ps PS 端，提供 DDR、GP master、HP0 slave、FCLK clock/reset。
AXI DMA axi_dma_0 MM2S 將 PS DDR input buffer 轉成 AXI4-Stream；S2MM 將 SR output stream 寫回 PS DDR。
SmartConnect smartconnect_mem 連接 AXI DMA 的 memory-mapped data ports 到 ZYNQ PS HP0。
Processor System Reset rst_ps7_0_100M 產生 peripheral reset / aresetn，提供 DMA 與 SR IP reset。
SR Accelerator custom IP sr_accel_axis_0 由 sr_accel_axis_ip_top package 出來的自訂 AXI4-Stream IP。

6. BMG ROM / RAM IP 與 COE 對應

Phase11.1 Tcl 會在 project 內重新建立 BMG IP，不直接搬移舊 project 的 XCI， 因為舊 XCI 的相對 COE 路徑容易失效。

IP module 型態 Width / Depth COE 用途
conv1_weight_rom Single Port ROM 8-bit / 72 model_lite/sr_core/generated_vivado_hex/conv1_weight.coe Conv1 3x3 Cout=8 weights。
conv3_weight_rom Single Port ROM 8-bit / 32 model_lite/sr_core/generated_vivado_hex/conv3_weight.coe Conv3 1x1 Cin=8/Cout=4 weights。
conv1_m0_rom Single Port ROM 32-bit / 8 model_lite/sr_core/generated_vivado_hex/conv1_m0.coe Conv1 per-channel M0。
conv1_m1_rom Single Port ROM 64-bit / 8 model_lite/sr_core/generated_vivado_hex/conv1_m1.coe Conv1 per-channel M1。
conv3_m0_rom Single Port ROM 32-bit / 4 model_lite/sr_core/generated_vivado_hex/conv3_m0.coe Conv3 per-channel M0。
conv3_m1_rom Single Port ROM 64-bit / 4 model_lite/sr_core/generated_vivado_hex/conv3_m1.coe Conv3 per-channel M1。
output_image_ram True Dual Port RAM 16-bit / 32768 無，runtime 寫入 Clean-stream core 內部 output image RAM，儲存 256x256 uint8 output 的 packed 16-bit 資料。

Vivado 產生的 XCI 會位於： model_lite/sr_core/PS_PL/vivado/sr_core_phase11_dma_pynqz2/sr_core_phase11_dma_pynqz2.srcs/sources_1/ip/。

8. 目前不在 Phase11.1 使用的東西

  • RTL_sys/phase9_6_clean_stream/：歷史/frozen source，不在 Phase11.1 直接引用。
  • vivado/sr_core_clean_stream_pynqz2/ 舊 project：是 Phase9.6 baseline，不是 Phase11.1 BD project。
  • Phase11.1 不使用 conv3_feature_ram；clean-stream 架構已移除該中間 RAM。
  • Phase11.1 尚未使用 AXI-Lite status/control register；目前只透過 DMA stream 搬 input/output frame。
  • Phase11.1 尚未執行 PYNQ Python runtime 或 board DMA smoke test。
