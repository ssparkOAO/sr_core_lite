Phase11.1 SR Accelerator IP 封裝與 AXI DMA Block Design

位置：model_lite/sr_core/PS_PL/html/phase11/phase11_1_dma_block_design.html

Phase11.1 開始進入 PS/PL system integration。本階段把 Phase10 已驗證的 AXI4-Stream wrapper 包成 Vivado IP，並在 PYNQ-Z2 Block Design 中接上 ZYNQ PS、AXI DMA、SmartConnect 與 reset logic。

1. 本階段目標

將 sr_core_axis_wrapper 包裝成適合 AXI DMA 使用的 AXI4-Stream IP。 Block Design 的資料流為： 

PS DDR input buffer
-> AXI DMA MM2S
-> AXI4-Stream
-> SR Accelerator IP
-> AXI4-Stream
-> AXI DMA S2MM
-> PS DDR output buffer

Phase11.1 不做 Python runtime、不做 AXI-Lite control/status、不做 HDMI/camera。 本階段只確認 IP packaging、BD 連線、validate、synthesis、implementation 與 bitstream 狀態。

2. Frozen Core 邊界

Phase9.6 clean-stream SR core 是 frozen baseline。Phase11.1 不修改 Conv1、Conv3、 PixelShuffle、OutputStage 或 clean-stream datapath，只新增外層 IP shell。 

sr_accel_axis_ip_top
-> sr_core_axis_wrapper
-> sr_top_clean_stream_img
-> Conv1 / Conv3 / PixelShuffle / OutputStage
-> output_image_ram

sr_accel_axis_ip_top 的任務是提供 Vivado IP Packager 容易辨識的 aclk、aresetn、S_AXIS、M_AXIS 與 tkeep。 內部把 active-low aresetn 轉成 wrapper 使用的 active-high rst。

3. AXI4-Stream 介面

介面 Port 說明
S_AXIS s_axis_tdata[7:0], s_axis_tkeep[0:0], s_axis_tvalid, s_axis_tready, s_axis_tlast AXI DMA MM2S 輸入 stream。每一拍是一個 8-bit pixel。
M_AXIS m_axis_tdata[7:0], m_axis_tkeep[0:0], m_axis_tvalid, m_axis_tready, m_axis_tlast 輸出 stream 送往 AXI DMA S2MM。m_axis_tkeep 固定為 1。
Clock aclk S_AXIS 與 M_AXIS 的 associated clock。
Reset aresetn Block Design active-low reset。內部使用 rst = ~aresetn。

4. Block Design 架構

ZYNQ PS
  | M_AXI_GP0
  v
AXI DMA control interface

ZYNQ PS DDR / HP0
  ^
  |
SmartConnect
  ^
  |
AXI DMA memory mapped ports

AXI DMA MM2S -> SR Accelerator S_AXIS
SR Accelerator M_AXIS -> AXI DMA S2MM

AXI DMA 的 MM2S 負責從 PS DDR 讀 input buffer 並轉成 stream。S2MM 負責把 SR accelerator 的 output stream 寫回 PS DDR。DMA 的 memory-mapped data ports 透過 smartconnect_mem 接到 ZYNQ PS 的 HP0 port。
