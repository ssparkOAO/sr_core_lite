回到 SR Core HTML GitHub Repository

Learning Note

我是怎麼把 sr_core 放到 GitHub，並變成線上 HTML 網站

這份頁面不是專案紀錄，而是給你複習用的操作筆記。目標是讓你知道每個按鈕、每個指令、每個網址背後的意思。

先說結論

Git

負責本機版本管理。它記住每次穩定修改，讓你可以回頭看、比較、復原。

GitHub

負責雲端備份與分享。把本機 Git repository 推上去後，其他電腦也能復原。

GitHub Pages

負責把 repository 裡的 HTML 檔案變成可以用瀏覽器開的網站。

這次實際做了什麼

  1. 選定 repository 根目錄

    我把 Git 管理範圍放在 model_lite/sr_core，因為這裡是核心 RTL、工具、HTML 文件所在的位置。

  2. 建立 .gitignore

    排除 Vivado 產生物、xsim 暫存檔、log、wdb、Python cache 和本機 backup。這可以避免 GitHub 塞滿不必要的大檔案。

  3. 初始化 Git 
    git init
git branch -M main
  4. 建立第一筆本機備份 
    git add .
git commit -m "Initial backup of SR core project"
  5. 接上 GitHub 遠端 repository 
    git remote add origin https://github.com/ssparkOAO/sr_core_lite.git
  6. 推送到 GitHub 
    git push -u origin main
  7. 開啟 GitHub Pages

    在 GitHub 的 Settings -> Pages 選擇 main branch 與 / root，GitHub 就會把 repository 根目錄當成網站來源。

為什麼網址一上去就是這個頁面？

你現在看到的網址是： 

https://ssparkoao.github.io/sr_core_lite/html/

它的意思可以拆成四段：

網址片段 代表意思
ssparkoao.github.io GitHub Pages 幫你的帳號建立的網站網域。
/sr_core_lite/ 你的 GitHub repository 名稱。
/html/ repository 裡面的 html 資料夾。
index.html 瀏覽器進入資料夾時，網站會自動找這個預設首頁。

所以 /html/ 其實就是打開 repository 裡的 html/index.html。你看到「SR Core 最小功能設計紀錄」，是因為那個檔案本來就是你的 HTML 入口頁。

以後你自己怎麼更新備份

每次完成一個穩定進度，例如 RTL 模擬通過、HTML 文件更新、Vivado Tcl 流程整理好，就做一次：

cd C:\M11413047\codex\final_proj\model_lite\sr_core
git status
git add .
git commit -m "Update phase8 ROM preload notes"
git push

重點不是每天固定 commit，而是每次「值得回復到這個狀態」時 commit。

三個常見情境

只想備份

做完穩定進度後，使用 git add、git commit、git push。這就是最基本也最重要的節奏。

想線上看 HTML

把 HTML 放在 repository 裡，GitHub Pages 會照資料夾路徑發布。資料夾裡有 index.html 時，網址可以省略檔名。

想大改但怕壞掉

先開分支，例如 git switch -c phase9-test。確認穩定後，再合回 main。

你需要記住的心法

  • GitHub 不是只拿來放檔案，它是用來保存每個穩定版本。
  • 不要把 Vivado 大量自動產生物都放進 Git，保留能重建專案的原始碼、設定與說明。
  • HTML 網站不是額外魔法，它只是 GitHub Pages 把 repository 裡的 HTML 檔案發布出去。
  • 每次 commit 訊息都要寫得像交接筆記，未來的你會感謝現在的你。
