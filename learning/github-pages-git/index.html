回到 SR Core HTML GitHub Repository

Learning Note

Git、GitHub、GitHub Pages 到底在做什麼

這份筆記用這次 `sr_core` 專案當例子，整理「本機備份」、「上傳 GitHub」和「HTML 變網站」之間的關係。

四個動作先背起來

add = 選檔案

git add 是把這次想存成版本的檔案放進準備區。它還不是存檔，也還沒上傳。

commit = 本機存檔點

git commit 是在你的電腦建立一個版本快照。以後可以比較、回頭看、復原。

push = 上傳 GitHub

git push 才是真的把本機 commit 上傳到 GitHub。GitHub 不會自己知道你本機改了什麼。

Pages = HTML 變網站

GitHub Pages 會把 repository 裡的 HTML、CSS、圖片發布成瀏覽器可以看的網站。

這次實際做了什麼

  1. 選定 Git 根目錄

    這次 Git 管理範圍放在 C:\M11413047\codex\final_proj\model_lite\sr_core。這個資料夾就是 GitHub repository 的根目錄。

  2. 建立 .gitignore

    排除 Vivado 產生物、xsim 暫存檔、log、wdb、Python cache 和本機 backup，避免把不必要的大檔案推上去。

  3. 初始化 Git 
    git init
git branch -M main
  4. 建立第一個本機存檔點 
    git add .
git commit -m "Initial backup of SR core project"
  5. 連到 GitHub repository 
    git remote add origin https://github.com/ssparkOAO/sr_core_lite.git
  6. 上傳 GitHub 
    git push -u origin main
  7. 開啟 GitHub Pages

    在 GitHub 的 Settings -> Pages 選擇 main branch 與 / root，GitHub 就會從 repository 根目錄發布網站。

為什麼 /html/ 會打開 html/index.html

你看到的網址是：

https://ssparkoao.github.io/sr_core_lite/html/

它的意思是：進入 sr_core_lite 這個網站裡的 html 資料夾。

當網址只指定到資料夾，沒有指定檔名時，網站伺服器會照慣例自動尋找這個資料夾裡的 index.html。 所以： 

https://ssparkoao.github.io/sr_core_lite/html/
=
https://ssparkoao.github.io/sr_core_lite/html/index.html

這不是因為資料夾裡只有一個 HTML。資料夾裡可以有很多 HTML，但只有 index.html 是預設首頁。

本機 / repository 路徑 線上網址
html/index.html /html/ 或 /html/index.html
html/phase8_index.html /html/phase8_index.html
learning/github-pages-git/index.html /learning/github-pages-git/

那這段指令在幹嘛

cd C:\M11413047\codex\final_proj\model_lite\sr_core
git status
git add .
git commit -m "Update phase8 ROM preload notes"
git push
指令 意思
cd ...\sr_core 進入 Git 根目錄，也就是這個 repository 的起點。
git status 看哪些檔案被修改、新增、刪除。
git add . 選擇目前資料夾底下的修改，準備放進下一個 commit。
git commit -m "..." 建立本機存檔點。引號裡是這次修改的簡短說明。
git push 把本機 commit 上傳到 GitHub。GitHub Pages 也會在之後依照新檔案更新網站。

什麼時候才要 push

你不需要做一點點就 push。建議等到一個「穩定段落」完成再推：

  • 一個 RTL module 或 testbench 跑通。
  • 一份 HTML 文件完成到可以給別人看。
  • Vivado / Tcl / 參數轉換流程確認能重現。
  • 一天工作結束，想留一份雲端備份。
  • 大改之前，先把目前可用狀態 commit + push。

怎麼知道我的網址有哪些

規則是把 repository 內的檔案路徑接到 GitHub Pages 網域後面：

https://ssparkoao.github.io/sr_core_lite/ + repository 裡的相對路徑

例子：

  • html/model_analysis.html -> https://ssparkoao.github.io/sr_core_lite/html/model_analysis.html
  • model.png -> https://ssparkoao.github.io/sr_core_lite/model.png
  • model_spec.png -> https://ssparkoao.github.io/sr_core_lite/model_spec.png
